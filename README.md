# AI-First Engineering Blueprint

My framework for building AI-first engineering organisations from the ground up. This body of work synthesises research from Anthropic's internal transformation study (132 engineers, 53 interviews, 200K agent transcripts), Cursor's engineering practices, and my own analysis of how these patterns apply to organisations building at the intersection of AI and security.

## What the framework covers

- **Primitive fluency** as the shared substrate for human-agent collaboration
- **The abstraction tax** framework for evaluating any tool or process
- **Security as first-class primitive**, not bolt-on
- **The harness pattern** for enabling long-running, multi-session agent work
- **Cultural foundations** for AI-native organisations
- **Skill development and the supervision paradox**
- **Risk-based delegation zones** (Green, Amber, Red, Human-Only)
- **Organisational structure and roles** for AI-first teams
- **12-month implementation roadmap** from foundation to maturity
- **Success metrics and KPIs**
